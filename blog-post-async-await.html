Async/Await vs Promises: Hangisini Kullanmalı?

Frontend Developer Senior Developer
Async/Await Promises JavaScript

JavaScript'te asenkron programlama için async/await ve Promises arasındaki farklar, avantajları ve hangi durumda hangisini kullanmanız gerektiği hakkında detaylı rehber.

Promises Nedir?

Promises, asenkron işlemlerin sonucunu temsil eden objelerdir. Bir işlemin başarılı olup olmadığını ve sonucunu handle etmenizi sağlar.

Promises Örneği 
function fetchUserData(userId) {
  return fetch(`/api/users/${userId}`)
    .then(response => {
      if (!response.ok) {
        throw new Error('Network response was not ok');
      }
      return response.json();
    })
    .then(data => {
      console.log('User data:', data);
      return data;
    })
    .catch(error => {
      console.error('Error:', error);
      throw error;
    });
}

Async/Await Nedir?

Async/await, Promises'i daha okunabilir ve senkron kod gibi yazmanızı sağlar. Async function'lar otomatik olarak Promise döndürür.

Async/Await Örneği 
async function fetchUserData(userId) {
  try {
    const response = await fetch(`/api/users/${userId}`);
    
    if (!response.ok) {
      throw new Error('Network response was not ok');
    }
    
    const data = await response.json();
    console.log('User data:', data);
    return data;
  } catch (error) {
    console.error('Error:', error);
    throw error;
  }
}

Avantajlar ve Dezavantajlar

Her iki yaklaşımın da kendine özgü avantajları ve dezavantajları vardır.

Promises Avantajları

  • Daha eski browser desteği
  • Chain yapısı ile akış kontrolü
  • Promise.all() ile paralel işlemler
  • Daha az memory kullanımı

Async/Await Avantajları

  • Daha okunabilir kod
  • Senkron kod gibi yazım
  • Daha kolay error handling
  • Debugging daha kolay

Hangi Durumda Hangisini Kullanmalı?

Proje gereksinimlerinize göre hangi yaklaşımı kullanacağınıza karar verebilirsiniz.

Kullanım Senaryoları

Async/await: Yeni projeler, okunabilirlik önemli
Promises: Eski browser desteği gerekli, chain yapısı tercih ediliyor

Paralel İşlemler

Birden fazla asenkron işlemi paralel olarak çalıştırmak için Promise.all() kullanabilirsiniz.

Paralel İşlemler 
// Promise.all ile paralel işlemler
async function fetchMultipleData() {
  try {
    const [users, posts, comments] = await Promise.all([
      fetch('/api/users').then(res => res.json()),
      fetch('/api/posts').then(res => res.json()),
      fetch('/api/comments').then(res => res.json())
    ]);
    
    return { users, posts, comments };
  } catch (error) {
    console.error('Error fetching data:', error);
    throw error;
  }
}

Sonuç

Async/await ve Promises, JavaScript'te asenkron programlama için güçlü araçlardır. Her ikisini de bilmek ve proje gereksinimlerinize göre seçim yapmak önemlidir.

Özet

  • Promises: Eski browser desteği, chain yapısı
  • Async/await: Okunabilirlik, kolay debugging
  • Her ikisini de bilmek önemli
  • Proje gereksinimlerine göre seçim yapın
  • Paralel işlemler için Promise.all() kullanın
