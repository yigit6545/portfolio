React 18'de Yeni Özellikler ve Concurrent Rendering

Frontend Developer Senior Developer
React 18 Concurrent Rendering Suspense Performance

React 18, frontend development dünyasında büyük bir devrim yarattı. Concurrent Rendering, Suspense ve Server Components gibi yeni özellikler ile React uygulamaları artık daha hızlı, daha responsive ve daha kullanıcı dostu.

Concurrent Rendering Nedir?

Concurrent Rendering, React'in en büyük yeniliklerinden biri. Bu özellik sayesinde React, render işlemini kesintiye uğratabilir ve daha önemli güncellemelere öncelik verebilir. Bu da kullanıcı deneyimini önemli ölçüde iyileştirir.

React 18 Concurrent Rendering Örneği 
import { createRoot } from 'react-dom/client';
import { StrictMode } from 'react';

// React 18'de yeni root API
const container = document.getElementById('root');
const root = createRoot(container);

root.render(
  <StrictMode>
    <App />
  </StrictMode>
);

Suspense ile Veri Fetching

React 18'de Suspense artık sadece lazy loading için değil, veri fetching için de kullanılabilir. Bu sayede loading state'lerini daha kolay yönetebiliriz.

Suspense ile Veri Fetching 
function UserProfile({ userId }) {
  const user = use(fetchUser(userId));
  
  return (
    <div>
      <h1>{user.name}</h1>
      <p>{user.email}</p>
    </div>
  );
}

function App() {
  return (
    <Suspense fallback={<Loading />}>
      <UserProfile userId={123} />
    </Suspense>
  );
}

Server Components

Server Components, React'in sunucu tarafında render edilebilen component'larıdır. Bu sayede bundle size'ı küçültebilir ve performansı artırabiliriz.

Önemli Not

Server Components henüz Next.js 13+ ile kullanılabilir durumda. Standalone React uygulamalarında henüz mevcut değil.

Automatic Batching

React 18'de state güncellemeleri otomatik olarak batch'lenir. Bu da gereksiz re-render'ları önler ve performansı artırır.

Automatic Batching Örneği 
function App() {
  const [count, setCount] = useState(0);
  const [flag, setFlag] = useState(false);

  function handleClick() {
    // React 18'de bu güncellemeler otomatik olarak batch'lenir
    setCount(c => c + 1);
    setFlag(f => !f);
    // Sadece bir re-render olur
  }

  return (
    <div>
      <button onClick={handleClick}>
        Count: {count}, Flag: {flag.toString()}
      </button>
    </div>
  );
}

useId Hook

useId hook'u, server ve client tarafında tutarlı ID'ler oluşturmak için kullanılır. Özellikle accessibility için önemlidir.

useId Hook Kullanımı 
function NameFields() {
  const id = useId();
  
  return (
    <div>
      <label htmlFor={`${id}-firstName`}>First Name</label>
      <input id={`${id}-firstName`} type="text" />
      
      <label htmlFor={`${id}-lastName`}>Last Name</label>
      <input id={`${id}-lastName`} type="text" />
    </div>
  );
}

Sonuç

React 18, frontend development'ı bir üst seviyeye taşıyor. Concurrent Rendering ile daha responsive uygulamalar, Suspense ile daha iyi loading state yönetimi ve Server Components ile daha iyi performans elde edebiliriz.

Özet

  • Concurrent Rendering ile daha responsive uygulamalar
  • Suspense ile gelişmiş veri fetching
  • Server Components ile daha iyi performans
  • Automatic Batching ile optimize edilmiş re-render'lar
  • useId ile tutarlı ID yönetimi
